Estimating with Use Cases

In 1993 Gustav Karner of Objectory AB did some research onto estimating man-hours for project completion based on use cases as a modification of estimation based on function points.

The following is a summary of Karner's work as it appears in [schneider1998].

(1) Weighted Actors
Total actor weight is the sum of each (actor * weight). AW = sum( actor * weight )
(2) Weighted Use Cases
Total use case weight is the sum of each (UC * weight) where use cases may be weighted by either transactions or analysis classes. UW = sum( use-case * weight )
(3) Unadjusted Use Case Points
UUCP = AW + UW
(4) Technical Complexity Factor
Technical factors are rated on a scale from 0 to 5. A rating of 0 means the factor is irrelevant to this project, 5 means it is essential. TFactor = sum( T * weight )
TCF = 0.6 + TFactor/100
(5) Environmental Factor
Environmental factors are rated on a scale from 0 to 5. See the environmental factor table below for rating details. EFactor = sum( E * weight )
EF = 1.4 - EFactor * 3/100
(6) Aggregate Use Case Points
UCP = UUCP * TCF * EF
(7) Project Estimate
Karner's original suggestion: man-hours = UCP * 20 hours
(8) Staff training and project stability refinement
Count E1-E6 that are less than 3. Count E7-E8 that are above 3. TR = sum( E1-E6 < 3 )
+ sum( E7-E8 > 3 )
if (2 < TR < 5) man-hours = UCP * 28 hours
if (5 <= TR) risk of project failure
(9) Additional Issues
Don't forget to
  • Add in time for communication (for large teams)
  • Add in time for working out team issues
